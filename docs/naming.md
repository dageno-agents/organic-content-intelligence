# Naming Notes

The original repository slug was `seo-content-funnel`.

That name is clear, but it undersells the system.

The project is not only an SEO funnel. It connects:

- organic search demand
- page-level behavior
- query and intent clusters
- content coverage
- brand knowledge freshness
- AI/GEO visibility
- data quality gates
- optimization task drafts

## Recommended Product Name

**Organic Content Intelligence**

Why:

- "Organic" covers SEO, organic search, and non-paid discovery.
- "Content" keeps the object clear.
- "Intelligence" signals diagnosis and decision support, not just reporting.
- The name leaves room for AI/GEO visibility and task drafting.

## Possible Repository Slugs

| Slug | Pros | Cons |
| --- | --- | --- |
| `organic-content-intelligence` | Accurate, high-level, expandable | Longer than the original slug |
| `content-growth-intelligence` | Strong product positioning | Slightly broader than the current technical scope |
| `organic-content-diagnostics` | Clear and developer-friendly | Less ambitious |
| `search-content-intelligence` | Good search signal | Weaker for AI/GEO and GA4 behavior |
| `seo-content-funnel` | Simple and concrete | Too narrow for the actual system |

## Current Recommendation

Use **Organic Content Intelligence** as the project title and `organic-content-intelligence` as the public repository slug.
